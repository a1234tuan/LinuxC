# 《软件工程》知识体系提纲

## 一、课程总体结构

| 模块                  | 作用与逻辑地位                                               |
| --------------------- | ------------------------------------------------------------ |
| **软件工程基础**      | 建立软件工程基本概念，理解软件危机背景，掌握软件生命周期和工程化思想 |
| **软件过程模型**      | 提供软件开发的组织框架，决定项目管理和技术活动的顺序与迭代方式 |
| **需求工程**          | 连接用户需求与系统设计的桥梁，明确系统"做什么"的问题         |
| **结构化分析与设计**  | 传统软件开发方法，通过功能分解和数据流分析构建系统模型       |
| **面向对象方法与UML** | 现代软件开发方法，通过对象建模支持系统演化与复用             |
| **面向对象分析**      | 识别问题域中的对象及其关系，建立系统概念模型                 |
| **软件设计模式**      | 提供经过验证的设计解决方案，提高软件设计的质量和复用性       |

## 二、章节级知识提炼

## 第1章 软件工程概述

### 核心知识点
- **软件概念与特性**
  - 软件定义：程序 + 数据 + 文档
  - 十大特性：形态、智能、开发、质量、生产、管理、环境、维护、废弃、应用
- **软件危机与工程化**
  - 软件危机的表现与原因
  - 工程化解决方案
- **软件生命周期**
  - 定义时期 → 开发时期 → 运行维护时期
  - 各阶段主要任务与产出物
- **软件工程方法学**
  - 结构化方法学（数据驱动）
  - 面向对象方法学（对象驱动）
  - 面向服务方法（接口驱动）
  - 面向数据方法（数据对象驱动）

### 前置依赖与关联关系
- 基础章节，为后续所有章节提供概念基础
- 软件生命周期概念贯穿整个课程体系

### 学习目标
- 能够解释软件工程的基本概念和必要性
- 能够描述软件生命周期的阶段划分及各阶段任务
- 能够区分不同的软件工程方法学及其适用场景

### 学习路径
1. 阅读Chapter01.ppt，理解基本概念
2. 观看"软件危机历史与影响"相关视频
3. 分析实际软件开发失败案例，理解工程化重要性
4. 对比不同规模项目的生命周期选择

### 达成标准
- 能清晰阐述软件工程的定义和目标
- 能画出软件生命周期图并解释各阶段任务
- 能举例说明不同软件工程方法学的特点

### 自测题
1. 软件与硬件的主要区别是什么？这对软件开发有什么影响？
2. 请用实际例子说明软件危机在现实项目中的表现。

### 推荐资源方向
- "软件工程基础概念"讲解视频
- 《软件工程(第9版) Sommerville》第1-2章

## 第2章 软件过程模型

### 核心知识点
- **瀑布模型**
  - 阶段间顺序性与依赖性
  - 推迟实现与质量保证观点
  - V模型变体
- **快速原型模型**
  - 抛弃型原型 vs 演化型原型
  - 用户需求验证机制
- **增量模型**
  - 增量构件开发
  - 优先级驱动的功能交付
- **螺旋模型**
  - 风险驱动的迭代过程
  - 四象限活动：目标设定、风险估计、开发验证、计划
- **统一过程(UP/RUP)**
  - 用例驱动、以架构为中心、迭代和增量
  - 四个阶段：初始、细化、构造、交付
- **敏捷过程**
  - 敏捷宣言与原则
  - 极限编程(XP)、自适应软件开发(ASD)

### 前置依赖与关联关系
- 基于第1章的软件生命周期概念
- 为后续需求分析、设计、测试等具体活动提供过程框架

### 学习目标
- 能够比较不同软件过程模型的优缺点
- 能够根据项目特点选择合适的开发模型
- 能够描述敏捷开发的核心价值观和实践

### 学习路径
1. 学习Chapter02.ppt，掌握各模型特点
2. 分析不同行业项目的案例，理解模型选择依据
3. 参与敏捷开发模拟练习
4. 比较传统与敏捷开发在需求变更处理上的差异

### 达成标准
- 能为给定项目场景推荐合适的软件过程模型
- 能解释迭代开发与增量开发的区别
- 能描述敏捷开发中"响应变化胜过遵循计划"的实际含义

### 自测题
1. 对于一个需求明确且变化较小的政府项目，你会选择哪种过程模型？为什么？
2. 请比较瀑布模型和螺旋模型在风险处理方面的差异。

### 推荐资源方向
- "软件过程模型对比分析"图表
- 敏捷开发实战案例视频

## 第3章 需求工程

### 核心知识点
- **需求工程过程**
  - 七个动作：起始、获取、细化、协商、规格说明、确认、管理
- **需求获取方法**
  - 用户分类与代表识别
  - 业务建模与系统流程图(SFD)
- **结构化分析技术**
  - 功能建模：数据流图(DFD)与环境图
  - 数据建模：实体-关系图(ERD)
  - 行为建模：状态转换图
- **需求规格说明**
  - 数据字典与加工规格说明
  - 决策表、决策树、结构化语言
- **需求验证与管理**
  - SRS质量特性：完整性、无歧义性、一致性、可验证性
  - 需求基线管理与变更控制

### 前置依赖与关联关系
- 基于软件过程模型确定的需求工程活动
- 输出为系统设计提供输入规格说明

### 学习目标
- 能够运用多种技术获取和分析用户需求
- 能够绘制分层DFD图并解释数据流逻辑
- 能够编写符合质量要求的软件需求规格说明

### 学习路径
1. 学习Chapter03.ppt，掌握需求工程技术
2. 使用Enterprise Architect或Visio绘制DFD、ERD图
3. 分析实际项目的SRS文档，理解质量要求
4. 参与需求访谈模拟，练习沟通技巧

### 达成标准
- 能为中小型系统绘制完整的分层DFD图
- 能编写清晰、无歧义的功能需求描述
- 能识别需求规格说明中的常见缺陷

### 自测题
1. 为图书馆管理系统绘制0层和1层数据流图
2. 写出"用户登录"功能的三个功能性需求和两个非功能性需求

### 推荐资源方向
- "需求分析实战技巧"视频教程
- IEEE 830标准：软件需求规格说明的建议实践

## 第4章 结构化设计方法

### 核心知识点
- **软件设计概念与原则**
  - 抽象、体系结构、模式、关注点分离
  - 模块化、信息隐藏、功能独立性
- **结构化设计过程**
  - 从DFD到结构图的映射
  - 变换型映射 vs 事务型映射
- **模块设计质量**
  - 耦合类型：数据耦合、控制耦合、公共耦合、内容耦合
  - 内聚类型：功能内聚、顺序内聚、通信内聚、过程内聚等
- **接口设计与数据设计**
  - 人机交互设计黄金规则
  - 文件设计与数据库设计选择

### 前置依赖与关联关系
- 基于第3章的需求规格说明
- 为后续的详细设计和实现提供架构基础

### 学习目标
- 能够将DFD转换为软件结构图
- 能够评估模块设计的质量（高内聚、低耦合）
- 能够应用软件设计原则改进系统架构

### 学习路径
1. 学习Chapter04.ppt，掌握结构化设计方法
2. 练习从DFD到结构图的转换
3. 分析现有系统的模块设计，评估耦合与内聚
4. 使用设计工具绘制结构图和接口规格

### 达成标准
- 能识别模块设计中的耦合和内聚问题
- 能应用变换分析或事务分析导出系统结构
- 能设计符合用户认知习惯的界面交互

### 自测题
1. 给定一个DFD图，将其转换为软件结构图
2. 分析某模块设计的耦合类型，提出改进建议

### 推荐资源方向
- "结构化设计实战"案例视频

## 第5章 面向对象方法与UML

### 核心知识点
- **面向对象基本概念**

- **UML建模语言**
  - 结构事物：类、接口、用例、组件等
  - 行为事物：交互、状态机
  - 分组事物：包
  - 注释事物
- **UML关系**
  - 依赖、关联、泛化、实现
  - 聚合与组合关系
- **UML图形**
  - 用例图、类图、交互图（顺序图、通信图）
  - 状态图、活动图、构件图、部署图

## 第6章 面向对象分析

### 核心知识点
- **面向对象分析过程**
  - 确定系统边界
  - 建立三种模型：用例模型、对象模型、动态模型
- **用例模型建立**
  - 识别业务参与者
  - 确定业务需求用例
  - 创建用例图与规格说明
- **对象模型建立**
  - 识别类与对象：实体类、边界类、控制类
  - 确定结构与关系：泛化、关联
  - 定义属性与服务
- **动态模型建立**
  - 顺序图：对象间交互的时间顺序
  - 状态图：对象生命周期状态变化

### 前置依赖与关联关系
- 基于第5章的UML建模知识
- 为面向对象设计提供分析模型

### 学习目标
- 能够从需求描述中识别对象和类
- 能够建立完整的OOA模型（用例模型+对象模型+动态模型）
- 能够应用分析模式解决常见领域问题

### 学习路径
1. 学习Chapter06.ppt，掌握OOA方法
2. 以选课系统为例，完整实践OOA过程
3. 比较结构化分析与面向对象分析的差异
4. 使用CASE工具支持OOA建模过程

### 达成标准
- 能针对具体问题域建立完整的OOA模型
- 能正确区分实体类、边界类和控制类
- 能使用状态图描述具有复杂状态行为的对象

### 自测题
1. 从银行ATM系统需求描述中识别主要对象和类
2. 为"订单"对象绘制状态图，描述从创建到完成的状态变迁

### 推荐资源方向
- "面向对象分析实战"项目案例
- 《面向对象分析与设计》经典教材

## 第11-2章 设计模式

### 核心知识点
- **设计模式概述**
  - 模式三层次：体系结构模式、设计模式、惯用语
  - 模式四要素：名称、问题、解决方案、效果
- **创建型模式**
  - 工厂方法、抽象工厂、单例、建造者、原型
- **结构型模式**
  - 适配器、装饰器、代理、外观、桥接、组合、享元
- **行为型模式**
  - 策略、模板方法、观察者、迭代器、责任链、命令等
- **设计原则**
  - 开闭原则、里氏代换、依赖倒转、接口隔离等

### 前置依赖与关联关系
- 基于面向对象设计与实现经验
- 提供经过验证的设计解决方案

### 学习目标
- 能够理解常见设计模式的意图和结构
- 能够在适当场景中应用设计模式解决问题
- 能够评估设计模式使用的利弊

### 学习路径
1. 学习设计模式.pdf，理解各模式特点
2. 分析开源框架中的设计模式应用
3. 重构现有代码，应用设计模式改进设计
4. 比较不同模式在相同问题上的解决方案

### 达成标准
- 能解释至少10种常用设计模式的适用场景
- 能在代码中识别和应用常见设计模式
- 能评估设计模式引入的复杂性与收益

### 自测题
1. 在什么场景下应该使用观察者模式？请举例说明
2. 单例模式的饿汉式和懒汉式实现各有什么优缺点？

### 推荐资源方向
- "设计模式实战解析"视频系列
- 《设计模式：可复用面向对象软件的基础》(GoF)

## 四、学习建议与注意事项

### ⚠️ 知识断层识别
1. **软件测试深度不足** - 建议补充测试策略、测试用例设计方法、自动化测试等内容
2. **软件维护专题缺失** - 建议学习软件演化、重构技术、遗留系统现代化等主题
3. **软件项目管理** - 建议补充项目估算、进度控制、风险管理、质量保证等管理主题
4. **软件配置管理** - 建议学习版本控制、变更管理、构建发布等工程实践

### 学习策略建议
1. **理论联系实际** - 每个概念都要尝试用实际项目例子理解
2. **工具技能培养** - 熟练掌握至少一种UML建模工具和IDE开发环境
3. **团队协作体验** - 通过小组项目体验软件工程中的沟通与协作
4. **持续学习意识** - 软件工程领域不断发展，保持对新方法技术的关注

### 实践项目建议
- 初级阶段：文档编写练习（SRS、设计文档）
- 中级阶段：小型系统分析与设计（如图书管理系统）
- 高级阶段：团队协作开发完整项目，体验完整软件生命周期

这份知识体系提纲基于你提供的PPT内容构建，涵盖了软件工程的核心知识领域。建议按照此框架系统学习，并结合实际项目经验深化理解。
