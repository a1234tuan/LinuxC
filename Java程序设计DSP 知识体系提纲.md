## 第1章 Java语言概述

### 核心知识点
- **Java语言特性**
  - 面向对象、平台无关、分布式、多线程、安全稳健
  - WORA（Write Once Run Anywhere）理念
- **Java平台组成**
  - JVM（Java虚拟机）：执行字节码的抽象计算机器
  - JRE（Java运行时环境）：包含JVM和核心类库
  - JDK（Java开发工具包）：包含JRE和开发工具
- **程序执行机制**
  - 编译型与解释型结合：javac编译 → JVM解释执行
  - 字节码与机器码的区别
- **开发环境配置**
  - 环境变量设置（JAVA_HOME, PATH）
  - 开发工具：文本编辑器、IDE（Eclipse, IDEA）

### 易混/易错概念
- JVM不是"Java虚拟机"的缩写，而是Java Virtual Machine
- JDK包含JRE，但JRE不包含开发工具
- 字节码不是机器码，需要JVM解释执行

## 第3章 面向对象程序设计（设计模式与内部类）

### 核心知识点
- **设计模式**
  - 单例模式：饿汉式、懒汉式、双重检查锁定、静态内部类实现
  - 策略模式：分离算法实现，支持运行时切换
  - 工厂模式：简单工厂、抽象工厂，封装对象创建过程
  - 回调模式：解耦调用方与被调用方，支持异步编程
- **内部类**
  - 成员内部类：关联外部类实例，可直接访问外部类成员
  - 静态嵌套类：不依赖外部类实例，不能直接访问外部类非静态成员
  - 局部内部类：定义在方法内，作用域受限
  - 匿名内部类：无类名，适用于一次性使用的场景
- **模拟多继承**：通过内部类继承多个类，解决Java单继承限制

### 易混/易错概念
- 单例模式线程安全问题：懒汉式需要同步，饿汉式类加载即创建
- 策略模式vs工厂模式：策略关注行为选择，工厂关注对象创建
- 内部类访问权限：非静态内部类隐含持有外部类引用
- 匿名内部类限制：不能同时继承类和实现接口

### 前置依赖
- 面向对象基础（封装、继承、多态）
- 接口与抽象类
- 基本Java语法和类定义

### 学习目标
- 能够识别并应用常见设计模式解决实际问题
- 能够根据场景选择合适的内部类类型
- 能够使用内部类模拟多继承场景

### 学习路径
1. 理解各设计模式的适用场景和实现原理
2. 通过对比学习掌握不同内部类的特性和用法
3. 完成设计模式小项目，如可扩展的日志系统

## 第6章 文件IO

### 核心知识点
- **File类**：文件和目录操作，不涉及内容读写
- **IO流分类**
  - 字节流：InputStream/OutputStream，处理二进制数据
  - 字符流：Reader/Writer，处理文本数据，支持编码转换
- **流类型**
  - 节点流：直接操作数据源，如FileInputStream、FileReader
  - 处理流：包装其他流，提供缓冲、过滤等功能
- **常用处理流**
  - 缓冲流：BufferedInputStream、BufferedReader，提高IO效率
  - 数据流：DataInputStream、DataOutputStream，读写基本数据类型
  - 对象流：ObjectInputStream、ObjectOutputStream，序列化对象
- **NIO简介**：Channel、Buffer的非阻塞IO

### 易混/易错概念
- 字节流与字符流的适用场景区别
- 节点流与处理流的装饰器模式关系
- 序列化要求类实现Serializable接口
- 字符编码问题：指定正确的字符集避免乱码

### 前置依赖
- 异常处理知识
- 集合框架基础

## 第8章 多线程编程

### 核心知识点
- **线程基础**
  - 进程vs线程：资源分配 vs 执行单元
  - 并发vs并行：逻辑同时 vs 物理同时
- **线程创建**
  - 继承Thread类，重写run()方法
  - 实现Runnable接口，作为Thread构造参数
  - Lambda表达式简化实现
- **线程生命周期**：新建、就绪、运行、阻塞、终止
- **线程控制**
  - sleep()：线程睡眠，不释放锁
  - yield()：让出CPU，重新竞争
  - join()：等待其他线程结束
  - interrupt()：中断线程
- **线程同步**
  - synchronized：同步方法、同步块
  - Lock接口：ReentrantLock，支持公平锁、条件变量
  - volatile：保证可见性，不保证原子性
- **线程通信**
  - wait()/notify()/notifyAll()：基于对象监视器
  - Condition：基于Lock的精确线程通信
- **线程池**：Executor框架，重用线程，降低资源消耗
- **生产者-消费者模式**：经典线程协作模型

### 易混/易错概念
- start() vs run()：启动线程 vs 普通方法调用
- synchronized锁定对象 vs 锁定类
- 死锁产生的四个必要条件
- 线程安全 vs 性能平衡

### 前置依赖
